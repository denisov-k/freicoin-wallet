# Freiland Registry — MVP-спецификация (релей-версия, без форка)

Второй столп Гезелля в крипте: дефицитные слоты (для MVP — **человекочитаемые
имена**) держатся не в вечной собственности, а «в аренду у общины». Рента — не
приватному лендлорду, а через демерредж. Никакого сквоттинга: у любого имени
всегда есть цена, по которой его можно выкупить.

Строится на существующем nv3 (user-токены + DEX + present-value + релей) с той же
моделью доверия, что и наши свопы: **релей индексирует и хостит доску, но не может
украсть** — токен и залог двигаются только подписями, которые кошелёк строит и
проверяет сам. Полная бесключевая (ковенант-)версия — отдельный софтфорк потом.

---

## 1. Одно число решает всё

Каждая регистрация описывается **единственной самозаявленной величиной `V`**
(в FRC). Из неё вытекает и цена, и рента:

- **`V` = цена принудительной продажи.** Любой платит `V` — забирает имя.
- **`V` = база ренты.** Держатель бондирует депозит ≈ `V`; демерредж этого депозита
  (≈5.09 %/год по консенсусу Freicoin) **и есть земельная рента**. Отдельного
  «платежа за аренду» нет — деньги «ржавеют» сами, ровно по Гезеллю.

Заявишь `V` низко — заплатишь мало ренты, но имя дёшево выкупят. Заявишь высоко —
имя защищено, но депозит тает быстрее. Гармония Харбергера, и вся рента —
«бесплатно» из уже работающего демерреджа. **Ставку ренты задаёт консенсус
(демерредж), не мы** — это ограничение, но оно же и есть гезеллевская ставка.

---

## 2. On-chain объекты (nv3, как есть)

1. **Имя = уникальный именованный токен.** nv3 уже умеет unique-named-items
   (токены, что не тают, ходят целиком, имена не повторяются — см. память
   *smart-property tokens*). Токен с именем `alice` = владение слотом `alice`.
   Держатель токена = держатель слота.
2. **Rent-депозит** — FRC, запертые в оффер-скрипте (наш ranged/HTLC-шаблон), так
   что (а) тают демерреджем (рента), (б) забираются покупателем атомарно при выкупе.
3. **Резолвер-запись** — куда указывает имя (адрес получателя). Хранится в witness-
   extension выхода токена (как assetTag-миграция уже делает) или в самом токене.

Депозит бондируется ≈ `V` (его годовой демерредж ≈ ставка × `V` = рента). Точная
формула привязки `депозит → V` (учёт уже стаявшего, порог делинквентности) —
единственная деталь бухгалтерии, которую надо зафиксировать в реализации.

---

## 3. Потоки

**Регистрация.** Выбрал свободное имя → задал `V` → выпустил именной токен (issue
на nv3) → запер депозит ≈ `V` → выставил на релее **постоянный оффер продажи**
токена по цене `V` (существующий ranged-оффер). Имя «зарегистрировано», пока живы
депозит и стоящий оффер.

**Держание.** Депозит тает (это рента). Present-value депозита сползает — значит и
твоя заявленная `V` сползает, если не пополнять. Забросил имя → его `V` и рента
дрейфуют вниз → его дёшево выкупают. **Сквоттеры вычищаются автоматически.** Хочешь
поднять цену/защиту — доливаешь депозит и перевыставляешь оффер на новую `V`.

**Принудительная продажа (forced sale).** Покупатель исполняет твой стоящий оффер:
атомарно платит `V` FRC → получает именной токен. Твой депозит разблокируется тебе
(минус стаявшее). Покупатель тут же регистрирует заново: своя `V`, свой депозит,
свой оффер. Это **буквально наш ranged-оффер/HTLC-своп**, где «активом» выступает
имя-токен. Никакого нового кода механики — переиспользование `core/dex.mjs` +
`swap-drive.mjs`.

**Резолв (польза уже сейчас).** Кошелёк резолвит `имя → адрес`: отправляешь на
`alice` вместо `fc1q…`. Это немедленная утилита (и повод пользоваться) поверх
чистой Freiland-механики.

## 4. Релей (market-server.mjs — доска, не хранилище ценности)

Новые read-модели поверх существующего p2p-стора:
- `landRegister({name, ownerPub, V, offerId, depositOutpoint, resolve})` — заявка
  регистрации (валидирует уникальность имени, наличие оффера и депозита).
- `landList / landLookup(name)` — индекс имён: держатель, `V`, стоящий оффер,
  present-value депозита, статус (живой / делинквентный).
- Выкуп идёт через уже существующие take/HTLC-эндпоинты DEX — отдельного «купить
  имя» не нужно, это ranged-тейк токена.

Доверие: релей может спрятать имя или оффер (цензура), но не подменить владельца —
токен и депозит двигаются только валидными подписями кошелька. Гарантия
«всегда-в-продаже» так же сильна, как то, что релей показывает стоящий оффер;
для MVP приемлемо (как весь наш DEX), апгрейд до ковенанта — потом.

## 5. Кошелёк (вкладка внутри существующего Биржа-флоу, по правилу UI-integration)

- **Найти/зарегистрировать имя** — поле имени, задать `V`, «Зарегистрировать»
  (issue токена + депозит + оффер одним визардом).
- **Мои имена** — список с `V`, present-value депозита, «долить»/«поднять цену».
- **Выкупить** — на чужом имени: «Купить за `V`» = ranged-тейк (уже есть).
- **Резолв** — в «Отправить»: ввёл `alice` → подставился адрес (переиспользует поле).

---

## 6. Что переиспользуем (≈70 % готово)

| Нужно | Уже есть в nv3 |
|---|---|
| Слот-маркер | unique-named tokens (issue, ходят целиком, имена уникальны) |
| Рента | демерредж + `assetPresentValue` (сам тает) |
| Всегда-в-продаже | ranged/pre-signed офферы (`core/dex.mjs`) |
| Атомарный выкуп | HTLC-своп механика (`swap-drive.mjs`) |
| Индекс/доска | релей p2p-стор + ордербук |
| UI | вкладка Биржа + форма «Отправить» (резолв) |

Нового: `land*`-эндпоинты релея (тонкая обёртка над p2p-стором), визард
регистрации в кошельке, резолвер `имя→адрес`, формула `депозит↔V`.

## 7. Явно вне MVP

- **Ковенант-версия** (forced-sale вшит в выход, без релея/пере-подписи) —
  консенсус-изменение на nv3, софтфорк; делать после доказанного спроса.
- **Demurrage-казна** (рента → прозрачный commons вместо майнеров) — тоже
  консенсус-изменение (меняет получателя вечной субсидии); отдельный, более
  тяжёлый шаг.
- Мульти-неймспейсы (тикеры, «участки», слоты) — та же механика, позже.
- Аукцион при первичной раздаче имён — можно добавить (nv3 умеет аукционы).

## 8. Порядок сборки

1. Формула `депозит↔V` + статус делинквентности (чистая функция + тесты).
2. `land*`-эндпоинты релея над p2p-стором; уникальность имён.
3. Визард регистрации в кошельке (issue+deposit+offer одной кнопкой).
4. Резолвер `имя→адрес` в «Отправить».
5. Выкуп = ranged-тейк токена (проводка существующего пути на имена).
6. «Мои имена» + долив/переоценка.

MVP = первый честный Freiland в крипте на готовой инфраструктуре, без форка.
Дистрибуционный крючок: «Gesell's Freiland + Harberger on-chain, живьём».
